# AGENTS.md

Руководство для AI-агентов по работе с MCP-сервером `putty-mcp` в этом репозитории.

## Базовый сценарий работы

1. Вызвать `putty_list_windows`.
2. Выбрать окно по `process_id` (предпочтительно) или `title_substring`.
3. При необходимости очистить старую историю: `putty_clear_window`.
4. Отправить команду: `putty_send_command`.
5. Подождать 1-2 секунды.
6. Считать вывод: `putty_read_window`.

## Правила выбора окна

- Если окон несколько, всегда указывать селектор (`process_id` или `title_substring`).
- Предпочитать `process_id` как самый надежный селектор.
- Если команда ушла не в то окно, повторить с явным `process_id`.

## Как уменьшать шум в контексте

- Перед диагностикой вызывать `putty_clear_window`, чтобы убрать длинный scrollback.
- Для сложных проверок использовать маркеры:
  - `echo DIAG_START`
  - команды диагностики
  - `echo DIAG_END`
- При разборе `putty_read_window` ориентироваться на участок между маркерами.
- Избегать длинных one-liner команд с тяжелым quoting; лучше 2-5 коротких команд.

## Ограничения

- Поддерживается только Windows (`pywin32` обязателен).
- `putty_send_command` подтверждает отправку, но не возвращает stdout команды.
- Фактический вывод читать только через `putty_read_window`.

## Поведение агента

- Не предполагать, что окно одно: сначала проверять `putty_list_windows`.
- Не выполнять потенциально опасные команды без явной просьбы пользователя.
- Для ответов пользователю возвращать краткую выжимку результата, а не весь сырой буфер.
